执行摘要

本报告深入调研 OpenClaw 与 Hermes Agent 两大 AI 个人助手平台的演进关系。Nous Research 官方将 Hermes 定位为"介于 Claude Code CLI 和 OpenClaw 消息平台代理之间"，明确承认两者共享 DNA。核心差异在于：OpenClaw 是成熟的"AI 助手平台"，强调集成编排；Hermes 是面向未来的"自进化 AI 框架"，具备闭环学习和自动技能创建能力。两者不是替代关系，而是互补演进。

二、演进关系定位

"Hermes Agent sits between a Claude Code style CLI and an OpenClaw style messaging platform agent"
— Nous Research 官方说明

这一定位明确承认 Hermes 和 OpenClaw 共享 DNA：两者都是个人 AI Agent，都支持 Telegram、Discord、Slack 等消息平台，都支持自托管，都有技能系统和持久化记忆。

演进视角：从"持久化"到"自进化"

代际代表产品核心能力关键差异
第一代OpenClaw跨平台消息集成、技能系统、持久记忆记忆存储事实，但不从经验创建技能
第二代Hermes Agent闭环学习、自动技能创建/改进、用户建模任务执行→学习→改进的循环

三、核心架构差异

1. 学习循环（Learning Loop）

OpenClaw：记忆系统跨会话存储事实，但不会从经验自动创建技能，依赖 MEMORY.md + AGENTS.md 手动管理。

Hermes Agent：情景记忆（ChromaDB 向量存储 + 语义检索），记录每个任务的执行，自动用于改进未来表现，"Closed Learning Loop" 真正自我改进。

五、选型建议

选择 Hermes Agent 当：

  • 需要自进化能力（Agent 自动从经验学习）
  • 需要 serverless 休眠（Daytona/Modal 零成本空闲）
  • 需要 Signal/Feishu 原生支持
  • 偏好 Python 开发

选择 OpenClaw 当：

  • 需要完整的平台生态（ClawHub 技能市场）
  • 需要移动端应用（iOS/Android/macOS 节点）
  • 需要Canvas 可视化（A2UI）
  • 需要复杂的多 Agent 编排（嵌套 subagent）
  • 偏好 TypeScript/Node.js

六、结论

两者不是替代关系，而是互补演进。OpenClaw 是成熟的"AI 助手平台"，Hermes 是面向未来的"自进化 AI 框架"。最佳实践是使用 OpenClaw 作为编排器（conductor）处理任务路由、认证管理、基础设施集成，使用 Hermes 作为专项专家（specialist）处理需要长期记忆和学习能力的专项任务。
