执行摘要

本报告深入调研 OpenClaw 与 Hermes Agent 两大 AI 个人助手平台的演进关系。Nous Research 官方将 Hermes 定位为"介于 Claude Code CLI 和 OpenClaw 消息平台代理之间"，明确承认两者共享 DNA。核心差异在于：OpenClaw 是成熟的"AI 助手平台"，强调集成编排；Hermes 是面向未来的"自进化 AI 框架"，具备闭环学习和自动技能创建能力。两者不是替代关系，而是互补演进。

一、基本概念定义

1.1 AI Agent（人工智能代理）

定义：能够感知环境、自主决策并执行行动以实现特定目标的智能系统。与传统 AI 工具不同，Agent 具备持续性（长期运行）、自主性（无需人工逐步指导）和工具使用能力（调用外部 API、操作文件系统等）。

核心特征：

  • 持久化记忆：跨会话保留上下文和学习成果
  • 工具编排：调用外部工具完成复杂任务
  • 多平台集成：连接消息平台、日历、邮件等
  • 自主规划：将复杂目标分解为可执行的子任务

1.2 个人 AI 助手（Personal AI Assistant）

定义：面向个人用户的 AI Agent，运行在用户自有设备上，直接访问用户的文件、消息、日历等私有数据，提供个性化的自动化服务。

与传统云 AI 的区别： 维度个人 AI 助手（OpenClaw/Hermes）云 AI（ChatGPT/Claude） 数据存储本地设备，用户完全控制云端服务器，厂商控制 系统集成深度访问文件系统、Shell、浏览器受限沙箱环境 持久性后台常驻，主动通知会话制，被动响应 隐私性数据不出本地数据上传云端

1.3 关键术语

术语定义
MCP（Model Context Protocol）Anthropic 提出的开放协议，标准化 LLM 与外部工具、数据源的集成方式
Skill（技能）Agent 的可扩展能力单元，通常包含 SKILL.md 定义文件和实现代码
Closed Learning Loop闭环学习：Agent 从任务执行中自动学习并改进自身行为
RAG（Retrieval-Augmented Generation）检索增强生成：从外部知识库检索信息来增强 LLM 输出
Subagent（子代理）由主 Agent 派生的专项 Agent，处理特定子任务
A2UIAgent-to-User Interface：Agent 与用户交互的可视化界面

二、发展时间线

2.1 OpenClaw 发展历程

2025年11月24日 — 诞生
Peter Steinberger 在马德里的周末发布了 "Clawdbot"，一个连接 AI 模型与消息应用的本地助手。首日获得 5,000 GitHub stars。

2026年1月27日 — 第一次更名
Anthropic 发出商标警告（"Clawd" 与 "Claude" 过于相似），项目紧急更名为 "Moltbot"。更名过程中 10 秒的空窗期被加密骗子利用，抢注了原 Twitter 账号并发行虚假 $CLAWD 代币。

2026年1月29-30日 — 正式定名 OpenClaw
更名为 "OpenClaw"，48 小时内获得 34,168 stars，创下 GitHub 最快增长纪录。

2026年2月2日 — 病毒式增长
达到 60,000 stars（仅用 3 天），登上 Hacker News 首页，获得 TechCrunch 等媒体报道。

2026年2月14日 — 创始人加入 OpenAI
Peter Steinberger 宣布加入 OpenAI 专注个人 Agent 研发，OpenClaw 项目移交给开源基金会管理。

2026年3月3日 — 超越 React
以 250,829 stars 超越 React（React 用了 10 年达到此数字），成为 GitHub 史上 stars 最多的项目。

2026年3月24日 — 历史新高
超过 335,000 stars，创历史最高纪录。

Nous Research 在 Discord 社区起步，发布 Hermes 1（基于 LLaMA 13B 微调），在多项基准测试中排名第一。

基于 Llama 3 70B 的 Hermes 3 80B 发布，登顶开源模型排行榜。获得 Paradigm 5000万美元 A 轮融资，代币估值 10 亿美元。

Nous Research 发布 Hermes Agent v0.1.0，从模型层向 Agent 层扩展。

2026年3月12日 — v0.2.0 首个正式版本
两周内合并 216 个 PR，解决 119 个 issue，新增 7 个消息平台、MCP 客户端支持、技能生态系统（70+ 技能）。

2026年3月23日 — v0.4.0 平台扩展
新增 6 个消息平台适配器（Signal、钉钉、Twilio SMS、Mattermost、Matrix、Webhook），4 个推理提供商。

2026年3月28日 — v0.5.0 安全加固
供应链安全审计，移除被攻击的 litellm 依赖，修复 50+ 安全漏洞。

2026年4月3日 — v0.7.0 弹性架构
可插拔内存提供程序、凭证池轮换、Camofox 反检测浏览器、168 个 PR 合并。

2026年4月8日 — v0.8.0 智能发布
后台任务自动通知、实时模型切换、MCP OAuth 2.1、209 个 PR 合并，达到 27,000 stars（242 名贡献者）。

四、演进关系定位

"Hermes Agent sits between a Claude Code style CLI and an OpenClaw style messaging platform agent"
— Nous Research 官方说明

4.1 官方定位对比

这一定位明确承认 Hermes 和 OpenClaw 共享 DNA：两者都是个人 AI Agent，都支持 Telegram、Discord、Slack 等消息平台，都支持自托管，都有技能系统和持久化记忆。

4.2 演进视角：从"持久化"到"自进化"

代际代表产品核心能力关键差异
第一代OpenClaw跨平台消息集成、技能系统、持久记忆记忆存储事实，但不从经验创建技能
第二代Hermes Agent闭环学习、自动技能创建/改进、用户建模任务执行→学习→改进的循环

五、核心架构差异

5.1 学习循环（Learning Loop）

OpenClaw：记忆系统跨会话存储事实，但不会从经验自动创建技能，依赖 MEMORY.md + AGENTS.md 手动管理。

Hermes Agent：情景记忆（ChromaDB 向量存储 + 语义检索），记录每个任务的执行，自动用于改进未来表现，"Closed Learning Loop" 真正自我改进。

七、选型建议

7.1 选择 Hermes Agent 当：

  • 需要自进化能力（Agent 自动从经验学习）
  • 需要 serverless 休眠（Daytona/Modal 零成本空闲）
  • 需要 Signal/Feishu 原生支持
  • 偏好 Python 开发

7.2 选择 OpenClaw 当：

  • 需要完整的平台生态（ClawHub 技能市场）
  • 需要移动端应用（iOS/Android/macOS 节点）
  • 需要Canvas 可视化（A2UI）
  • 需要复杂的多 Agent 编排（嵌套 subagent）
  • 偏好 TypeScript/Node.js

八、结论

两者不是替代关系，而是互补演进。OpenClaw 是成熟的"AI 助手平台"，Hermes 是面向未来的"自进化 AI 框架"。最佳实践是使用 OpenClaw 作为编排器（conductor）处理任务路由、认证管理、基础设施集成，使用 Hermes 作为专项专家（specialist）处理需要长期记忆和学习能力的专项任务。
